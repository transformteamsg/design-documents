# Design Documents

This repository contains engineering-wide and product-specific architectural design documents, following a structured process using **Request for Comments (RFCs)** and **Architecture Decision Records (ADRs)**.

## Purpose

- **RFCs** are used to propose and discuss significant architectural or product decisions.
- **ADRs** document the final decisions and their context for future reference.

## Structure

- **Engineering-wide ADRs:**  
  Stored in the [`adr/`](./adr/) directory.
- **Product-specific ADRs:**  
  Each product has its own directory at the root (e.g., `/product-foo/`).

All ADRs use a 4-digit numbering scheme and a descriptive filename, e.g., `0001-standardise-rfc-and-adr-templates.md`.

For more details, see the [ADR 0001](./adr/0001-standardise-rfc-and-adr-templates.md).
